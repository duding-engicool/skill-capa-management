# CAPA纠正预防措施管理技能（capa-management）

> 类型：交互引导式（成熟度定制）｜ 输出：txt + md

## 一句话简介
按组织成熟度（A 草根 / B 基础 / C 体系）定制「审核书面合规 + 现场轻量化」双轨落地方案，强制 3 问判定成熟度、六步不跳步、六段式输出可直接递交审核的纠正措施报告。

## 适用角色
质量工程师、CAPA 负责人、品质经理、过程改进工程师、中小企业负责人。

## 何时使用
- 处理客诉 / 审核不符合项 / 过程异常，需要正式纠正措施（CAPA）；
- 组织管理基础薄弱，套用体系模板写不出也落不了地；
- 认证 / 客户审核要求整改闭环材料，既要书面合规、又要现场真能执行；
- 重复性、重大质量问题需要把"措施—验证—标准化"落到纸面。

## 核心机制
- **强制前置成熟度判定**：首次对话先问 3 个通俗问题（现场干活靠什么 / 出问题怎么管 / 应对审核靠什么），判定 A/B/C 类组织；
- **双轨策略库**：按成熟度匹配「审核书面合规材料 + 现场轻量化落地动作」，杜绝纸面整改；
- **六步不跳步**：锁定事实 → 应急纠正 → 根因分析 → 定制措施 → 审核闭环 → 防反弹固化；
- **六段式输出**：事实描述 / 应急纠正 / 根因(5Why) / 分轨措施 / 效果验证 / 防反弹固化。

## 产出物
| 文件 | 说明 |
|------|------|
| `CAPA纠正措施报告_{编号}_{日期}.md` | 标准六段式文档（可直接打印递交审核） |
| `CAPA纠正措施报告_{编号}_{日期}.txt` | 同内容纯文本版（对齐排版，便于归档/邮件） |

## 使用方式
1. 在对话中按引导先回答成熟度判定的 3 个问题；
2. 逐项提供不符合事实 / 应急纠正 / 根因 / 措施 / 验证 / 固化信息；
3. 确认大纲；
4. Agent 调用 `scripts/build_report.py` 生成 txt + md 双件（默认写入当前工作目录）。

本地脚本跑通（内置小样本）：
```bash
python scripts/build_report.py --demo --out-dir ./输出
```
或用自己的结构化 JSON：
```bash
python scripts/build_report.py --input result.json --out-dir ./输出
```

## 设计规范
- 所有文档简体中文；代码标识符英文；
- 防幻觉：不确定项标「待企业补充」，不编造标准号 / 数据；
- 去商业化：无营销人设、无付费引导；
- 输出为内部 / 正式审核文档，默认 txt + md（不出网页）；如用于投屏评审汇报再补网页版。
